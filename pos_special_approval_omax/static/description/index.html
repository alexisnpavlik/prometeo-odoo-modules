POS Manager Special Approval

OMAX INFORMATICS

Overview

This module delivers an advanced POS manager validation system with multiple authentication methods including barcode scan, password, and PIN code authentication. Secure your POS operations with configurable validation triggers for critical actions like discounts, price changes, payments, and refunds.

Features

  • Multiple Authentication Methods (Barcode, Password, PIN)
  • Virtual keyboard support for secure password entry
  • One-Time Password per Order feature
  • Configurable validation triggers for POS operations
  • Manager validation for order deletion and modifications
  • Discount and price change approvals
  • Payment and refund processing controls
  • Cash in/out operation validations
  • Selective payment method validations
  • Add Product and Quantity Change validations
  • Interactive POS popups with camera scan functionality

How It Works

POS Settings Configuration


Point of Sale → Configuration → Settings → POS Manager Validation

Configuration Options

  • Enable/Disable: Complete POS Manager Validation feature control
  • POS Managers: Select internal users for validation
  • Validation Type: All methods, Barcode only, Password only, PIN only, or combinations
  • Virtual Keyboard: Optional secure password entry interface
  • One Time Password for Order: Enter Password once per order
  • Payment Methods: Specific payment methods requiring validation
  • Validation Triggers: add product to order, order deletion, order line deletion, discounts, price changes, quantity changes, payments, refunds, cash operations


User Credentials Setup


Settings → Users → Manage Users → Select User → Point Of Sale Validations


Validation Operations


POS Operations Validation

The following operations trigger validation popups based on your configuration settings:

  • Add Product to Order: Manager validation when adding products to the order
  • Quantity Change: Validation required when modifying product quantities
  • Apply Discount: Manager approval needed for discount applications
  • Change Price: Price modification validation
  • Remove Order Line: Validation for order line deletion
  • On Payment: Payment processing validation
  • On Cash Move: Cash in/out operations validation

Note: All validation functionalities work consistently with external keyboard inputs as well as on-screen interactions.



Order Delete Validation


Order Deletion Control

Manager validation is required before deleting entire orders.



Payment Method Validation


Selected Payment Method Validation

Validation is triggered when specific payment methods are selected or when payment amounts are entered directly.

Note: If price is entered directly without selecting a payment method, and the default payment method is configured for validation, the validation will still be triggered automatically.



Refund Validation


Refund Processing Control

Manager approval is required for processing refunds to ensure proper authorization and prevent unauthorized refund processing.



Authentication Methods


Important Notes:
• The validation methods are set according to the configuration, and if only one method is selected, the "Choose Validation Method" popup does not appear.
• For all validations, the popup layout looks the same - only the header changes based on the validation type.
• From all configured methods, any one authentication is required to proceed.
• The screenshots of the payment popups are shown below.



Barcode Authentication

Scan the manager's barcode for quick authentication using the camera scan functionality on camera-supported devices.



Password Authentication

Secure password entry with optional virtual keyboard for enhanced security and touch-screen compatibility.



PIN Code Authentication

Numeric PIN entry for quick and secure manager authentication with dedicated PIN pad interface.



Wrong Validation

If entered credentials are wrong then this popup appears.



One-Time Password Configuration


One-Time Password Setting

Enable this option to authenticate once per order and skip repeated validation for the same manager on that order, improving workflow efficiency while maintaining security.
